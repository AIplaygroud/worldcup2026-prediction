# V3.8 四场第二轮赛后审计与规则修订（2026-06-23）

## 一、审计结论

四场赛后结果表明，V3.8 对全场方向的识别表现较好，但样本量仅为 4 场，不能据此认定模型已经充分校准，更不能直接提高强队系数、EventFlow 权重或大比分尾部参数。

- 胜平负最高概率方向：4/4 命中。
- 大小 2.5 方向：4/4 命中。
- BTTS 方向：4/4 命中。
- 实际比分进入概率派 Top-5：3/4；法国 3-0 为 Top-1，挪威 3-2 排第 9。
- 四场平均实际赛果预测概率：0.6157。
- 三分类平均 Brier：0.2643；平均 Log loss：0.5181。
- 半场胜平负 / 半全场的主排序与实际过程吻合度偏低，当前不具备与全场胜平负同等级的置信度。

因此，V3.8 当前应被评价为：

> **全场概率主干具有可用性；精确比分和事件路径有一定信息价值；半场/半全场仍属于低置信度试验层；工程链路存在若干必须修复的问题。**

## 二、概率上限规则修订

### 2.1 取消“85%胜率铁律”

原 README / skill 中规定任何球队胜率不得超过 85%。该规则属于人为固定截断，会造成以下问题：

1. 极端强弱对阵即使数据和历史校准支持更高概率，也被机械压缩；
2. 截断后的概率不一定保持良好校准；
3. 文档规定与实际程序输出不一致，例如法国胜率实际输出 89.39%；
4. 固定上限不能替代真正的概率校准。

新版规则为：

- 取消 85% 固定上限；
- 若数据完整、模型可靠且历史校准支持，胜率可以超过 85%；
- 任何胜、平、负概率都不得达到 100% 或 0%；
- 数值实现采用极小 ε 进行开区间保护，而不是设置影响正常结果的固定上限；
- 极端概率必须同时报告数据覆盖、校准状态、模型可靠度和主要反向情景；
- 不得把高概率写成“必胜”“稳胆”或无风险结论。

### 2.2 推荐实现

保留两列概率：

- `raw_probability`：模型原始概率；
- `calibrated_probability`：基于时间外历史样本校准后的概率。

最终发布概率使用校准值，并执行：

```text
p_final = clip(p_calibrated, epsilon, 1 - epsilon)
```

其中 epsilon 仅防止数值上出现 0 或 1，不构成新的 99%、95% 或 85%人为上限。

## 三、半场胜平负与半全场置信度修订

### 3.1 为什么当前命中能力偏低

半场与半全场比全场胜平负更难稳定预测，原因包括：

- 45 分钟样本短，单个进球影响极大；
- 点球、红牌、伤退和门将失误会快速改变状态；
- 教练中场调整无法完全从赛前静态特征推断；
- 两段泊松对上下半场进球分配的假设仍较简化；
- EventFlow 的半场输出目前主要是场景排序，不是充分校准概率；
- 四场中，阿根廷、法国、约旦—阿尔及利亚的实际半场路径均未由第一排序稳定捕捉。

因此，不能因为半全场赔率高，就把排名第一的组合视为可直接投注的概率结论。

### 3.2 新的输出纪律

预测结果中：

- 半场胜平负 / 半全场可以继续输出；
- 必须单列为“低置信度参考”；
- 必须给出多个候选分布，避免单点断言；
- 必须注明主要敏感因素；
- 数据不足或引擎降级时允许不输出。

投注策略中：

- 半场胜平负 / 半全场不得进入稳健层；
- 不得作为单关主仓、保底项或串关唯一胆项；
- 默认只允许进入高风险小注候选；
- 必须标注“低置信度、事件敏感、非核心”；
- 只有完成独立时间外回测并达到预设校准门槛后，才允许提升为均衡层。

### 3.3 后续校准指标

半场和半全场应独立评估：

- 半场胜平负：Brier、Log loss、RPS；
- 半全场九分类：multiclass Brier、Log loss、Top-1 / Top-3 命中率；
- 校准曲线：可靠性图、ECE；
- 分层表现：强弱悬殊、均势、早球高风险、低位防守、必须抢分、天气异常；
- 与简单基线对照：首半场泊松、全场概率折算、市场无抽水概率。

在样本不足前，不根据单轮比赛调整上下半场分配系数。

## 四、四场预测与实际过程对照

### 4.1 阿根廷 2-0 奥地利

- 全场赛果、低总进球、BTTS 否和 2-0 比分带判断正确。
- 实际 2-0 位于概率派 Top-3。
- 半场实际为阿根廷领先，但 EventFlow 的半全场第一排序偏向平局路径，说明早球分配不稳定。
- 结论：全场结构正确；半场层低置信度；阿根廷机会量可能被低估。

### 4.2 法国 3-0 伊拉克

- 3-0 为概率派 Top-1，赛果、零封和早球方向均正确。
- 法国胜率 89.39% 不再被视为违反规则；新版允许超过 85%。
- 但总 λ 偏高，大于 3.5 球概率可能过热。
- 结论：不应重新设置 85%上限，而应检验极端概率校准和大比分尾部校准。

### 4.3 挪威 3-2 塞内加尔

- 挪威胜、BTTS 是和大于 2.5 球全部正确。
- 实际 3-2 仅排第 9，表明比分尾部仍有低估可能。
- 但补时进球、伤退和比赛状态突变具有较强随机性，不能因单场直接提高尾部参数。
- 结论：保留现有参数，加入滚动时间外样本观察。

### 4.4 约旦 1-2 阿尔及利亚

- 阿尔及利亚胜和 1-2 精确比分命中。
- 实际过程为约旦半场领先、阿尔及利亚下半场逆转，半全场第一排序未捕捉。
- 两粒阿尔及利亚进球来自定位球，现有 EventFlow 对替补调整、定位球和逆转路径刻画不足。
- 结论：全场概率主干正确；半场/半全场不稳定；定位球和替补深度值得做通用模块，而不是单场规则。

## 五、必须修复的链路问题

### 5.1 J44 主客比分映射错误

EventFlow 在客队为优势方时没有镜像比分族，曾把客队优势比分错误输出为主队 2-0、2-1。该问题属于确定性语义错误，必须修复并增加镜像单元测试。

### 5.2 V3.7 加载失败仍继续输出

J44 出现 `loaded=false`、`quality=degraded`，但融合仍生成结果。应做到：

- 失败层权重自动归零；
- 最终可靠度同步下调；
- 报告顶部明确显示降级状态；
- 禁止把“执行过脚本”写成“特征有效参与”。

### 5.3 场景权重完整性不足

J44 初次运行缺少 EventFlow 场景权重，需要手工注入。应在运行前完成比赛映射、球队画像、matchup、场景、V3.7 特征和输出路径的完整性预检。

### 5.4 共享表临时替换

运行 R2 时临时替换 processed 表会带来状态污染、并发冲突和复现风险。应改为每场独立 run 目录、显式输入清单、文件哈希和代码版本记录。

### 5.5 数据质量口径误导

“已有数据 100%真实”不等于“数据完整”。建议拆分为：

- authenticity：来源真实性；
- coverage：字段覆盖率；
- freshness：数据时效；
- consistency：多源一致性。

最终可靠度应明显受 coverage 和 consistency 约束。

### 5.6 半场层缺少独立降级门控

即使半场/半全场数据不足，当前报告仍会常规输出排序。应增加：

- `halftime_confidence`；
- `halftime_calibration_status`；
- `halftime_data_coverage`；
- 低于门槛时自动标记低置信度或不输出投注候选。

## 六、不应根据四场立即调参

- 不因四场全中提高融合权重或强队 λ；
- 不因法国胜率超过 85%重新设置固定硬上限；
- 不因挪威 3-2 直接提高大比分尾部；
- 不因阿尔及利亚两次角球得分加入单场专用规则；
- 不因半全场连续未命中就手工反转某些路径权重；
- 不根据赛后结果重新筛选当场证据。

所有参数调整必须建立在冻结版本、时间切分、独立测试集和基线比较基础上。

## 七、不过拟合的改进路线

### 7.1 立即修复，不需要重新拟合

1. 修复主客比分镜像；
2. 完善引擎加载与降级门控；
3. 删除 85%固定上限，增加概率开区间保护；
4. 增加半场/半全场低置信度标签；
5. 增加场景和数据完整性预检；
6. 隔离每场运行目录；
7. 细分数据质量与冲突计数。

### 7.2 需要扩大样本后再决定

1. 定位球进攻与防守模块；
2. 替补席实力和中场调整模块；
3. 强队落后后的逆转状态树；
4. 必须抢分比赛的大比分尾部；
5. 上下半场进球分配与半全场校准。

### 7.3 验证流程

1. 冻结当前 V3.8 核心参数；
2. 使用后续小组赛和淘汰赛作为时间外测试；
3. 同时评估简单 Poisson、概率派、双引擎融合和市场无抽水概率；
4. 全场与半场分开评估，禁止用全场表现替代半场校准；
5. 使用 bootstrap 置信区间检验改进是否稳定；
6. 只有多个时间窗口持续改善，才允许调整权重或提升半场投注等级。

---

## 八、当前版本的最终定位

V3.8 当前可以继续用于：

- 全场胜平负方向；
- 比分概率带；
- 总进球和 BTTS；
- EventFlow 的战术路径解释与排序辅助。

当前不应高置信使用：

- 半场胜平负；
- 半全场九分类；
- 依赖单一早球路径的投注；
- 引擎降级后仍输出的高置信结论。

最终原则：

> **允许高概率，但不允许确定性；允许半场参考，但不允许把未校准的半场排序包装成高置信概率。**
